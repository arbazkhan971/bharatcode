# db

**Path:** `internal/db/`
**Status:** Completed

## Purpose

The `db` package owns BharatCode's on-disk state. It opens a single SQLite database at `~/.local/share/bharatcode/bharatcode.db` (or `$XDG_DATA_HOME/bharatcode/bharatcode.db` when set), applies embedded schema migrations on open, exposes a `*sql.DB` for ad-hoc queries, and surfaces a `*sqlc.Queries` value containing type-safe CRUD generated by [sqlc](https://sqlc.dev) from SQL files in `internal/db/sql/`.

It exists so every other module (`session`, `message`, `filetracker`, `ledger`, `config` overrides) writes through one schema-versioned store with WAL-mode concurrency and CGO-free deployment. The driver is `modernc.org/sqlite`, a pure-Go translation of SQLite that links cleanly under `CGO_ENABLED=0`. No raw `sql.DB` handles escape this package; callers receive `*DB`, which composes `*sql.DB` and `*sqlc.Queries`.

## Public interface

```go
// Package db owns BharatCode's SQLite persistence layer. The schema is
// defined as SQL migrations under internal/db/migrations/ and
// query code is generated by sqlc into internal/db/sqlc/.
package db

import (
    "context"
    "database/sql"

    sqlc "github.com/<org>/bharatcode/internal/db/sqlc"
)

// DB is a handle to BharatCode's persistent store. It wraps the
// underlying *sql.DB and exposes a *sqlc.Queries for generated
// type-safe access. The zero value is not usable; callers must call
// Open.
type DB struct {
    *sql.DB
    Queries *sqlc.Queries
}

// Open opens (or creates) the SQLite database at path, applies pending
// migrations, configures pragmas (WAL, foreign keys, busy_timeout),
// and returns a ready-to-use *DB. The parent directory is created with
// 0o755 if it does not exist. Open is safe to call concurrently for
// different paths; concurrent calls for the same path are serialized
// internally and return distinct *DB values sharing the underlying
// *sql.DB connection pool.
func Open(ctx context.Context, path string) (*DB, error)

// Close closes the underlying *sql.DB. Subsequent calls to any method
// on the returned *DB return ErrClosed. Close is idempotent.
func (db *DB) Close() error

// Migrate applies any pending migrations from the embedded migrations
// filesystem. It is called automatically by Open and exposed for tests
// and the `bharatcode migrate` CLI command. Migrate is idempotent:
// calling it twice on the same database is a no-op on the second call.
func (db *DB) Migrate(ctx context.Context) error

// Path returns the absolute filesystem path of the database file.
func (db *DB) Path() string

// ErrClosed is returned by methods on a *DB after Close has been called.
var ErrClosed = errors.New("db: connection closed")
```

## Schema

Initial migration `internal/db/migrations/001_initial.sql` declares the tables below. All timestamps are stored as unix milliseconds (`INTEGER NOT NULL`). All foreign keys cascade on delete.

```sql
-- +goose Up
-- +goose StatementBegin

CREATE TABLE sessions (
    id            TEXT    PRIMARY KEY,
    project_path  TEXT    NOT NULL,
    title         TEXT    NOT NULL,
    model         TEXT    NOT NULL,
    agent         TEXT    NOT NULL,
    created_at    INTEGER NOT NULL,
    updated_at    INTEGER NOT NULL
);

CREATE INDEX idx_sessions_project_path ON sessions (project_path);
CREATE INDEX idx_sessions_updated_at   ON sessions (updated_at DESC);

CREATE TABLE messages (
    id            TEXT    PRIMARY KEY,
    session_id    TEXT    NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    role          TEXT    NOT NULL CHECK (role IN ('system','user','assistant','tool')),
    content_json  TEXT    NOT NULL,
    parent_id     TEXT             REFERENCES messages(id) ON DELETE SET NULL,
    created_at    INTEGER NOT NULL
);

CREATE INDEX idx_messages_session_id ON messages (session_id);
CREATE INDEX idx_messages_parent_id  ON messages (parent_id);

CREATE TABLE file_changes (
    id            TEXT    PRIMARY KEY,
    session_id    TEXT    NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    path          TEXT    NOT NULL,
    operation     TEXT    NOT NULL CHECK (operation IN ('create','update','delete','rename')),
    before_hash   TEXT,
    after_hash    TEXT,
    created_at    INTEGER NOT NULL
);

CREATE INDEX idx_file_changes_session_id ON file_changes (session_id);
CREATE INDEX idx_file_changes_path       ON file_changes (path);

CREATE TABLE ledger_entries (
    id             TEXT    PRIMARY KEY,
    session_id     TEXT    NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    provider       TEXT    NOT NULL,
    model          TEXT    NOT NULL,
    input_tokens   INTEGER NOT NULL CHECK (input_tokens >= 0),
    output_tokens  INTEGER NOT NULL CHECK (output_tokens >= 0),
    cost_usd       REAL    NOT NULL CHECK (cost_usd >= 0),
    cost_inr       REAL    NOT NULL CHECK (cost_inr >= 0),
    created_at     INTEGER NOT NULL
);

CREATE INDEX idx_ledger_entries_session_id ON ledger_entries (session_id);
CREATE INDEX idx_ledger_entries_created_at ON ledger_entries (created_at);

CREATE TABLE config_kv (
    key    TEXT NOT NULL,
    value  TEXT NOT NULL,
    scope  TEXT NOT NULL CHECK (scope IN ('global','project')),
    PRIMARY KEY (scope, key)
);

-- +goose StatementEnd

-- +goose Down
-- +goose StatementBegin
DROP TABLE IF EXISTS config_kv;
DROP TABLE IF EXISTS ledger_entries;
DROP TABLE IF EXISTS file_changes;
DROP TABLE IF EXISTS messages;
DROP TABLE IF EXISTS sessions;
-- +goose StatementEnd
```

Pragmas applied on every connection-pool acquisition (idempotent):

| Pragma | Value | Why |
|---|---|---|
| `journal_mode` | `WAL` | Concurrent readers + single writer without blocking. |
| `foreign_keys` | `ON` | Cascades only fire when this is on; off by default in SQLite. |
| `busy_timeout` | `5000` | 5-second wait for the writer lock before returning SQLITE_BUSY. |
| `synchronous` | `NORMAL` | WAL-safe; faster than FULL with no durability hit short of OS crash. |
| `temp_store` | `MEMORY` | Faster sorts/joins; database stays small. |

## Dependencies

- `internal/util` — `util.ExpandPath` resolves `~` and env vars in the configured database path; `util/fsext.EnsureDir` creates the parent directory.
- External: `modernc.org/sqlite` — pure-Go SQLite driver, registered as `"sqlite"`.
- External: none for migration execution; migrations are read from `embed.FS`
  and applied by a package-local runner.
- Build-time only: `github.com/sqlc-dev/sqlc/cmd/sqlc` — invoked by `scripts/sqlc-generate.sh` to regenerate `internal/db/sqlc/` from `internal/db/sql/*.sql` plus the migration set. The generated code itself depends only on stdlib `database/sql`.
- stdlib: `database/sql`, `embed`, `context`, `errors`, `path/filepath`, `sync`, `time`.

14. `go build ./...` succeeds with `CGO_ENABLED=0`.

## Notes for the implementer

- Generated code lives in a sub-package: `internal/db/sqlc/`. Set `sqlc.yaml` `gen.go.package: "sqlc"` and `gen.go.out: "internal/db/sqlc"`. Hand-written files (`open.go`, `migrate.go`, `db.go`) sit in `internal/db/` and import `sqlc "github.com/<org>/bharatcode/internal/db/sqlc"`. Do NOT place generated code next to hand-written code under the same package — it makes coverage exclusion noisy.
- `sqlc.yaml` lives at repo root. Schema source is the union of `internal/db/migrations/*.sql`; query source is `internal/db/sql/*.sql`. Engine: `sqlite`. Emit `emit_interface: true`, `emit_json_tags: true` (snake_case), `emit_pointers_for_null_types: true`.
- Migrations are embedded via `//go:embed migrations/*.sql` on a package-level `embed.FS`. A package-local runner reads the `-- +goose Up` section markers for compatibility with the existing migration files and records applied filenames in `schema_migrations`.
- Set `db.SetMaxOpenConns(1)` on the underlying `*sql.DB` to serialise writes and avoid WAL header desync under concurrent sub-agents.
- Pragmas are set via a `connector` wrapper or applied after `sql.Open` in a loop before the first query. Both work; the connector is cleaner because every new pool connection picks them up automatically.
- `Open` must `db.PingContext(ctx)` before running migrations, so a corrupt file fails fast instead of mid-migration.
- All errors wrapped with `fmt.Errorf("opening db at %s: %w", path, err)` style messages (per AGENTS.md §4).
- Tests use `t.TempDir()`, `t.Setenv()`, `github.com/stretchr/testify/require`. Never hit a real DB outside the temp dir. Run benchmarks under `go test -bench=.` not `-run`.
- Foreign-key enforcement must be re-asserted after every connection acquisition: `PRAGMA foreign_keys` is per-connection in SQLite, and the modernc driver's connection pool may hand out fresh connections that revert to the default.
- The `config_kv` table is *not* the primary config store — that is the JSON files owned by `internal/config/`. `config_kv` exists for runtime override flags written by `bharatcode set …` commands. Document this in a comment over the table definition.
