# Shell

**Path:** `internal/shell/`
**Status:** Completed

## Purpose

Execute bash commands on behalf of the agent and the user, with timeouts, working-directory control, environment overrides, separate capture of stdout and stderr, and a background-job tracker. Synchronous `Run` blocks until completion and returns the full `Job`; asynchronous `Start` returns immediately with a job id that the tool layer (and the user) can use to poll status, fetch output, or kill the job. Streaming output is published to a `pubsub.Topic[JobUpdate]` so the TUI can render live logs without polling. This module is the substrate for the `bash` tool and for the `hooks` module's command runner.

## Public interface

```go
type Shell struct{ /* unexported */ }

func New(bus *pubsub.Topic[JobUpdate]) *Shell

func (s *Shell) Run(ctx context.Context, cmd string, opts RunOpts) (Job, error)
func (s *Shell) Start(ctx context.Context, cmd string, opts RunOpts) (jobID string, err error)
func (s *Shell) Kill(jobID string) error
func (s *Shell) Output(jobID string) (Job, error)

type RunOpts struct {
    Cwd     string
    Timeout time.Duration
    Env     map[string]string
}

type Job struct {
    ID        string
    Command   string
    StartedAt time.Time
    Status    JobStatus
    ExitCode  int
    Stdout    string
    Stderr    string
}

type JobStatus string // Running, Completed, Failed, Killed, Timeout

type JobUpdate struct {
    JobID  string
    Stream string // "stdout" or "stderr"
    Chunk  []byte
    Done   bool
}
```

## Dependencies

Internal: `util`, `pubsub`.
External: stdlib `os/exec`, stdlib `syscall` (for process groups on Unix).

## Acceptance criteria

- `Run` with `Timeout` set returns `Status == Timeout` and a killed process when the deadline elapses; no zombies survive.
- Killing a parent shell terminates its descendants on Unix (verified by spawning `bash -c 'sleep 60 & wait'` and confirming the sleep is gone).
- A command producing 10 MB+ of stdout completes without OOM; output is truncated past the cap with a clear marker (`... [truncated, N bytes]`).
- Concurrent jobs are fully isolated: distinct `ID`s, distinct cwd, no shared file descriptors, race-free state under `go test -race`.
- `Output` on a still-running job returns the current accumulated stdout/stderr snapshot without disturbing the running process.
- `Kill` on a completed job is a no-op that returns `nil` (idempotent).

## Notes for the implementer

Execute every command as `bash -c <command>` so users get pipes, redirections, globbing, env expansion, and `&&` without us reimplementing a shell parser. On platforms without bash (Windows), document the gap rather than emulating — Phase 1 is Unix-first.

On Unix, set `SysProcAttr.Setpgid = true` and `Pgid = 0` so the child gets its own process group; `Kill` then signals the negative pgid to deliver to every descendant. Without process-group isolation, background jobs the script started survive the parent kill and become orphan zombies.

Cap captured `Stdout` and `Stderr` at 10 MB each. Wrap the read pipe in a counting writer that stops appending past the cap but keeps draining the pipe (closing it early would SIGPIPE the child). Emit a trailing `... [truncated, N bytes]` marker so the agent and the user can tell the truncation happened.

Job ids are short random strings (8 hex chars is plenty) generated by `crypto/rand`; do not use sequential ids that leak ordering. Store jobs in an `sync.Map` keyed by id, with a separate ttl-based cleanup goroutine that drops `Completed`/`Failed`/`Killed`/`Timeout` jobs older than 10 minutes — otherwise long sessions leak memory.

Timeouts use `context.WithTimeout` derived from the caller's context; do not start a side goroutine with `time.AfterFunc` (it leaks if the command finishes early). The `RunOpts.Env` map is merged onto `os.Environ()`, not replaced — users almost always want to add `PATH` entries, not erase the whole environment.

## Implementation status

- **Status**: Completed.
- **Implemented features**:
  - Full synchronous (`Run`) and asynchronous (`Start`) execution using Unix-compliant `bash -c`.
  - Robust process group termination targeting negative PGID (`syscall.Kill(-pgid, syscall.SIGKILL)`) to eliminate any orphan background zombies.
  - Efficient, low-overhead pipe reading up to 10 MB caps per stream, falling back to a clean draining cycle with the descriptive `... [truncated, N bytes]` message when caps are exceeded.
  - Safe concurrent execution verified under `-race` detector.
  - Memory-efficient Sync.Map registry with a 1-minute TTL check evicting finished commands older than 10 minutes.
- **Deviations**: None. Everything is built to perfect parity with specifications.
